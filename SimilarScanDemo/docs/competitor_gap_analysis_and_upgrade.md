# 竞品扫描差距分析与 Demo 改造说明

## 1. 结论

原 Demo 复用了竞品的 `dHash + colorHash + 阈值`，但在资源分类、结果互斥和视频指纹上加入了竞品没有的自定义规则，因此扫描数量和结果分组与竞品存在差异。

本次改造优先解决直接造成漏扫和漏展示的问题。

## 2. 竞品支持的结果分类

竞品源码定义了以下分类：

- 相似照片
- 重复照片
- 视频
- 相似截图
- 其他截图
- 全部截图
- 相似视频
- 其他图片
- 聊天图片
- 相似录屏
- 其他录屏

竞品媒体类型包括普通图片、视频、截图和录屏，不包含音频扫描。

## 3. Demo 当前支持

- 相似普通照片
- 相同照片和截图
- 相似截图
- 相似视频和相似录屏
- 其他视频和录屏
- 其他截图
- 其他普通图片
- 音频列表和基础疑似重复
- SQLite 指纹缓存和结果分组
- 分批扫描和扫描中结果刷新
- 分类详情两列网格
- 大图预览和左右切换

## 4. 本次已完成改造

### 4.0 录屏验证后的产品结构调整

根据 `screen-20260622-101228.mp4` 的首次授权与完整扫描流程，Demo 已取消原来的横向 tab，改为竞品一致的纵向分类模块：

- Similar
- Duplicates
- Similar Screenshots
- Similar Videos
- Other Screenshots
- Chat Photos
- Similar Screen Rec
- Other Screen Rec
- Other Videos
- Other

首页每个分类显示资源数量、容量和最多三张预览图。数量为 0 的分类仍保留入口。

相似类详情按算法组分段，每组内部使用两列布局；普通分类直接使用两列网格。单个媒体仍可进入大图预览并左右切换。

### 4.0.1 首次启动和订阅展示

Demo 增加三个可区分的首次启动引导页：

1. 本地隐私和相册访问说明。
2. 重复照片分组清理。
3. 释放手机存储空间。

第三页之后进入独立订阅展示页。订阅页不接真实支付，关闭按钮和试用按钮都进入扫描首页。

完成首次流程后通过 SharedPreferences 记录状态，后续启动直接进入首页。

### 4.1 分媒体权限隔离

图片、视频和音频分别检查权限。

单个媒体类型没有权限或 MediaStore 查询失败时，只跳过该类型，不中断其他类型扫描。

Android 14 及以上继续区分完整访问和部分照片访问。

竞品本身只扫描图片和视频，因此 Demo 的竞品对齐链路不再申请或扫描音频权限。

### 4.2 资源结果不再被隐藏

- 取消相似组最多 80 组的产品展示限制。
- 取消“其他”分类最多 1000 条的限制。
- 缩略图或指纹生成失败的资源继续进入“其他”分类。
- 音频 tab 展示全部音频结果。
- 重复视频在没有独立重复视频 tab 时继续归入其他视频。

### 4.3 图片候选召回修正

删除原来的 `dHash` 高 16 位数值相邻过滤。

该过滤与汉明距离没有等价关系，会在正式阈值比较之前误删大量候选。

当前改为：

```text
同媒体类型
+ 相近宽高比
-> 完整 dHash / colorHash 阈值判断
```

该方案优先保证召回率。后续面对 10 万级媒体库，可进一步使用 BK-Tree 或 MIH 优化比较性能。

### 4.4 视频多帧匹配

进一步追踪反编译调用链后确认，图片使用 `CGAsset.b()` 缩略图路径，视频则使用
独立的 `MediaMetadataRetriever` 多帧路径。Demo v12 改为：

```text
按时长抽取 7～13 帧
-> 每帧生成 dHash + colorHash
-> 所有帧交叉比较
-> 至少两次命中后判定相似
```

同时删除 400MB 拒扫条件。CSV 中 436,512,117 字节的视频此前因此标记为 FAILED，
直接造成 Other Videos 多出一条。

### 4.5 重扫一致性

重扫单个资源前会移除它的旧分组关系，扫描完成后清理成员不足两个的失效分组，避免历史结果残留。

候选查询同时排除资源自身。

### 4.6 录屏识别

除原有的 `screenrecord`、`screen recording`、`录屏` 外，增加：

- `screen-record`
- `recording`
- `capture`
- `mirror`
- `cast`

### 4.7 聊天图片分类

从媒体文件名、相册名和相对路径识别以下常见聊天来源：

- WhatsApp
- Telegram
- Messenger
- WeChat / Weixin
- LINE
- Viber
- Signal

### 4.8 推荐保留和删除交互

- 每个相似组按照分辨率、文件大小和创建时间选择一项作为 `Best`。
- 默认保留 `Best`，并选中同组其他资源。
- 支持单项选择、全选和取消全选。
- 底部实时显示选中数量和预计释放容量。
- Android 11 及以上通过 `MediaStore.createDeleteRequest` 弹出系统删除确认。
- 删除完成后同步清理 SQLite 中的资源、指纹和分组关系。

## 5. 仍需验证或外部系统支持

- 真实订阅计费、恢复购买和服务端权益校验。
- 多品牌真机、10 万级真实媒体库的召回率和误判率验收。
- HDR、HEIF、损坏媒体和极端长视频的设备兼容数据。
- Android 后台限制下的长期周期任务；当前已使用前台服务承载用户触发扫描。

## 6. 数据库升级

数据库版本升级到 8，重建旧版多帧视频指纹和历史互斥关系错误的分组。

安装新版 APK 后旧数据库会重建，需要重新执行完整扫描。
